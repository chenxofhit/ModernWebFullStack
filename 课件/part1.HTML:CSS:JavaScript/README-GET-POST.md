# GET 和 POST 方法对比演示

## 简介

这是一个简单易懂的示例，用于展示HTTP GET和POST方法的主要区别。

## 文件说明

- `get-post-demo.html` - 演示页面，包含GET和POST表单
- `server.js` - 简单的Node.js服务器，用于处理请求并展示区别

## 使用方法

### 方法1：使用Node.js服务器（推荐）

1. 确保已安装Node.js
2. 在终端中运行：
   ```bash
   node server.js
   ```
3. 打开浏览器访问：`http://localhost:3000/get-post-demo.html`
4. 分别提交GET和POST表单，观察区别

### 方法2：直接打开HTML文件

1. 直接用浏览器打开 `get-post-demo.html`
2. 表单提交会使用JavaScript拦截并展示区别（不会真正发送到服务器）

## GET 和 POST 的主要区别

| 特性 | GET | POST |
|------|-----|------|
| **数据位置** | URL查询参数 | 请求体（Body） |
| **可见性** | 数据在URL中可见 | 数据在请求体中，URL中不可见 |
| **安全性** | 较低，数据暴露在URL中 | 较高，适合传输敏感信息 |
| **数据长度** | 受URL长度限制（通常2048字符） | 理论上无限制 |
| **用途** | 获取数据（查询） | 提交/修改数据 |
| **缓存** | 可以被浏览器缓存 | 通常不被缓存 |
| **书签** | 可以保存为书签 | 不可以 |
| **幂等性** | 幂等（多次请求结果相同） | 非幂等（可能产生副作用） |

## 实际应用建议

- **使用GET的场景：**
  - 搜索查询
  - 筛选/过滤数据
  - 获取资源列表
  - 不需要修改服务器状态的操作

- **使用POST的场景：**
  - 用户登录（密码等敏感信息）
  - 提交表单数据
  - 创建/更新/删除资源
  - 上传文件
  - 任何会修改服务器状态的操作

## 注意事项

⚠️ **重要：** 虽然POST比GET更安全，但HTTPS才是真正保护数据传输安全的方式。无论是GET还是POST，在不加密的HTTP连接中，数据都可能被截获。
